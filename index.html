文件与目录

在项目里移动、创建、复制与清理文件的基础动作。

查看当前目录

显示你当前所在的绝对路径。

pwd

列出文件详情

显示隐藏文件、权限、大小与修改时间。

ls -la

进入目录

使用 .. 可返回上一级目录。

cd path/to/directory

创建目录

-p 会同时创建不存在的父目录。

mkdir -p src/components

创建空文件

文件存在时会更新它的修改时间。

touch README.md

复制文件或目录

复制目录时需要加 -r。

cp -r source-dir backup-dir

移动或重命名

同一条命令可用于移动文件和修改文件名。

mv old-name new-name

删除文件

删除目录请使用 rm -r directory，务必先确认路径。

rm filename

搜索与查看

快速定位代码、查看内容，少在目录树里迷路。

读取文件内容

适合简短文本；大文件可配合 less 使用。

cat package.json

分页查看文件

按 q 退出，输入 /关键词 可搜索。

less README.md

查看文件末尾日志

-f 会持续跟踪新增内容，常用于看日志。

tail -f app.log

搜索代码内容

推荐使用 rg（ripgrep），速度快且默认忽略常见无关目录。

rg "TODO" src

按名称查找文件

在当前目录递归寻找匹配的文件名。

find . -name "*.test.js"

查看磁盘空间

-h 用易读单位显示各磁盘使用情况。

df -h

进程与网络

排查端口占用、定位进程和测试服务连通性。

查看指定端口占用

找出正在监听 3000 端口的进程。

lsof -i :3000

结束进程

先尝试普通结束；无响应时才使用 -9。

kill <pid>

按名称查找进程

查找包含关键字的进程，例如 node 或 vite。

ps aux | rg node

测试 HTTP 响应

仅显示响应头，适合检查接口状态码与重定向。

curl -I https://example.com

查看本机 IP

macOS 上查看 Wi-Fi 网卡的本地 IPv4 地址。

ipconfig getifaddr en0

实时查看资源占用

按 CPU、内存等指标查看正在运行的进程。

top

环境与权限

检查命令来源、管理环境变量，以及处理常见的权限问题。

查看命令位置

确认当前实际执行的是哪个版本的命令。

which node

查看环境变量

读取一个环境变量的值。

echo $PATH

临时设置环境变量

只在当前终端会话中生效。

export NODE_ENV=development

查看文件权限

权限位如 rwx 分别对应读、写与执行。

ls -l script.sh

添加可执行权限

让 Shell 脚本能够直接运行。

chmod +x script.sh

查看命令历史

配合 history | rg git 可过滤历史记录。

history

Git 版本控制

下面保留完整的 Git 日常操作速查，便于在同一页完成开发现场的大多数查询。

开始与查看

先弄清当前状态，再开始操作，是很好的 Git 肌肉记忆。

初始化仓库

在当前项目目录创建一个新的 Git 仓库。

git init

查看工作区状态

查看改动、暂存区状态和当前分支。

git status

查看改动内容

显示尚未暂存的文件差异。

git diff

查看暂存区差异

显示已 add、但还没提交的改动。

git diff --staged

分支协作

让每个功能或修复在独立分支上生长。

查看本地与远程分支

-a 会同时显示远程分支。

git branch -a

新建并切换分支

从当前提交创建功能分支并立即进入。

git switch -c feature/login

切换已有分支

切回主分支或任意本地已有分支。

git switch main

合并一个分支

先切到接收改动的分支，再执行合并。

git merge feature/login

删除已合并分支

安全删除本地已合并的分支。

git branch -d feature/login

更新远程分支信息

获取远程最新引用，并清理已删除的远程分支。

git fetch --prune

提交与同步

小而完整的提交，未来的你会感谢现在的你。

暂存指定文件

把一个文件的改动加入下一次提交。

git add src/app.js

暂存全部改动

将当前目录下的新建、修改、删除全部加入暂存区。

git add .

创建提交

写一条清晰说明本次改动意图的提交信息。

git commit -m "feat: add login page"

修改最近一次提交

补充漏掉的改动，或修改最近一次提交说明。

git commit --amend

拉取并合并远程改动

获取远程更新并合并到当前分支。

git pull origin main

推送当前分支

首次推送用 -u 建立追踪关系。

git push -u origin feature/login

撤销与救援

动手前先确认范围；这里的几条是开发现场的“后悔药”。

取消暂存文件

保留工作区内容，只把文件移出暂存区。

git restore --staged src/app.js

丢弃未暂存的文件改动

将文件恢复为最近一次提交的版本。

git restore src/app.js

反做一个提交

生成一个新提交来撤销指定提交，适合已推送的历史。

git revert <commit-id>

找回误操作前的提交

查看 HEAD 移动记录，找回 reset、rebase 前的提交。

git reflog

暂存工作

临时切去修一个紧急问题，未完成工作也能妥善收好。

临时保存当前改动

保存已跟踪文件的改动；-u 同时包含未跟踪文件。

git stash -u

查看暂存列表

列出所有暂存起来的工作现场。

git stash list

恢复最近的暂存

恢复并删除最近一份 stash。

git stash pop

标签与日志

给版本落点命名，也给历史留下一张清晰的地图。

简洁查看提交历史

以单行图形方式查看分支和提交关系。

git log --oneline --graph --all

创建附注标签

通常用于发布版本，例如 v1.0.0。

git tag -a v1.0.0 -m "release v1.0.0"

推送标签

将单个标签同步到远程仓库。

git push origin v1.0.0

⚠️ 需要格外小心的操作

rm 会直接删除文件；git restore 会丢弃未提交的改动；git push --force 会重写远程历史。执行前请确认路径或分支，备份重要工作，并优先使用 --force-with-lease。
